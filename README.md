# resumo_labDIO_02
Este repositório contém o resumo das lições aprendidas durante o desenvolvimento do lab na DIO


Aqui está o resumo do laboratório:

No laboratório, exploramos vários aspectos relacionados ao **Service Level Agreement (SLA)** no Azure:

1. **Tabela do SLA e Uptime garantido**: Vimos como interpretar a tabela de SLA, que indica o tempo de atividade garantido (Uptime) para cada serviço. Quanto mais altos os valores do SLA, maior o tempo de disponibilidade prometido.

2. **Ações em caso de não cumprimento do SLA**: Estudamos o que acontece quando a Microsoft não cumpre o SLA, incluindo políticas de compensação ou créditos para o cliente.

3. **Escolha do melhor SLA para o serviço**: Aprendemos a identificar o SLA mais adequado dependendo do tipo de serviço e suas necessidades de disponibilidade.

4. **Importância dos "9's" no SLA**: Entendemos que quanto mais "9's" no número do SLA (por exemplo, 99,99%), maior é o tempo de disponibilidade garantido para o serviço.

5. **Navegação para o Docs via ícone de informação**: Exploramos como acessar a documentação do Azure diretamente no portal por meio do ícone de informação, facilitando a obtenção de detalhes sobre os serviços.

6. **Estratégias de replicação e zonas de disponibilidade**: Aprendemos sobre as estratégias de replicação, como o uso de zonas de disponibilidade para garantir maior resiliência e disponibilidade dos serviços.

7. **Impacto das replicações no SLA e no custo**: Vimos como essas estratégias de replicação aumentam o SLA, mas também podem influenciar no custo, exigindo um equilíbrio entre alta disponibilidade e o orçamento disponível.

Esse laboratório focou em entender como o SLA afeta a disponibilidade dos serviços, as opções de replicação e como esses fatores influenciam tanto o desempenho quanto o custo operacional no Azure.
